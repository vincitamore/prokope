# Lens: formal-statement-first

Before hunting a proof, force the claim into a fully formal statement — machine-checkable, every object defined, every quantifier explicit. What resists formalization is usually the actual gap, and the definitions the formalization demands are themselves discoveries.

## Questions that fire it

- Can you state the claim so precisely that a proof assistant (or a referee with no goodwill) accepts the statement as well-formed — and if not, which object is the one you cannot yet define?
- What does the formal statement's context require that your informal work has been borrowing silently (a measure, a limit's existence, a domain restriction, an implicit uniformity)?
- Does formalizing a small already-proved piece expose reusable structure the prose proof hid?

## Overfire (vice)

- Formalization as a substitute for attack — a perfectly stated open problem is still open.
- Stub theater: placeholder statements that formalize the label rather than the claim.
