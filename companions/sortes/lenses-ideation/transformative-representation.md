# Lens: transformative-representation

When an object cannot be directly represented or precisely quantified in the current system, extend or transform the system so the object becomes first-class and operable — do not force limit-only talk or refuse the object.

If the object is not an element of S that can participate in S's operations without resorting only to limiting processes, construct S′ where direct representation and quantification become possible. Methodological, not axiomatic; may add structural cost.

## Questions that fire it

- Is the target object missing from the domain of the current system, or only reachable by limits / indirect talk rather than as an operable element?
- What minimal extension S′ would make that object first-class (new coordinates, completion, compactification, dual chart, enlarged number system) — and what operations then become algebraic that were asymptotic before?
- Does the extension earn itself by a new exhibit path, or is it ornament that still only renames limits?

## Overfire (vice)

- Extending S when the object already lives cleanly in S (unnecessary machinery).
- Using S′ as escape from hard work without a falsifiable payoff.
- Confusing pure transport (rename of classical facts) with a transformative representation that actually hosts new operations.
