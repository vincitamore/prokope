# Lens: inherited-partition

Re-examine the question-space cut inherited from prior work or founders — answering only inside it can miss the productive recut.

## Questions that fire it

- Which partition of the problem did we inherit (tracks, basins, “allowed” methods), and who set it?
- What question cannot even be asked inside that partition, and would it matter if opened?
- Is the next move forced by the inheritance, or have we verified the cut still matches the operation-space we need?
