# Lens: change-the-engine

Name the generative engine class of your current model — independent trials, composition/product structure, spectral data, walk dynamics, filtration, extremal principle, constraint solver — and check it against the ledger of engines already tried. If the class is already priced, the next variant is too: build the next attempt from an engine class not yet on the ledger.

## Questions that fire it

- What is the engine actually generating your current model's predictions — and is it isomorphic, under relabeling, to an engine already tried under a different name?
- Which engine classes has the whole effort actually exercised, and which classes (dependence structures, algebraic actions, geometric flows, extremal principles) have zero entries on the ledger?
- What is the cheapest construction that genuinely leaves the current class — not a reparametrization, but a model whose failure would be news given everything tried so far?

## Overfire (vice)

- Class-hopping without depth — an engine class is not exhausted by one shallow probe.
- Taxonomy for its own sake: the classification pays only when it changes what gets built next.
