# Collapse catalog — reference

Failure modes of long-horizon goal loops. Use for diagnosis; the list is not exhaustive — add modes as your own goals exhibit them.

| Mode | One-line |
|------|----------|
| Count-collapse | N iterations / pipelines / reports treated as an exit ticket |
| Volume ≠ altitude | Commits, nodes, and files accumulate with no density gain and no new engine |
| Kill-pressure death | Falsification rate trends to zero while the status honestly stays "open" |
| False decorrelation | The sampler's draws vary; the behavior freezes |
| One-engine costume | A single generative model re-skinned under many vocabularies |
| Adversary sleep | Adversarial review is optional at first, then silently absent |
| Trail lapse | Draw-tagging and `not surfaced:` hygiene abandoned mid-marathon |
| Packaging elevation | Substrate or a change of variables sold as a summit claim |
| Tip lag | Sessions resume from a stale status head instead of the true tip |
| Surface bloat | Tracking surfaces outgrow their orientation value; rotate eras — freeze the old as forensics, open fresh artifacts |
| Overfit gospel | One campaign's constants promoted to universal law |
| Soft-KEEP theater | Intermediate pins accumulate until they read as altitude |
| Obstruction museum | Cataloguing what blocks progress without ever changing the engine |
| Construction overfire | Building forever without kill criteria on any of it |
| Summary-as-warrant | Compaction/summary text treated as exhibit for bars or claims |
| Ghost continuity | Standing blocks, draws, or next moves assumed still held in chat after a boundary |
| Tip-erasure | A boundary accepted without a tip write |
| Re-constitution from summary | Doctrine or the bar rewritten from blurred memory |
| Soft-elevation after compaction | Summary "progress" read as altitude |
| Orientation inversion | Whole-corpus read before the tip on re-entry |
| Boundary thrash | Compaction → soft continue → compaction, without a tip or a kill |
| Recurrent bar-falsification | The operator falsifies a self-declared true success; the loop re-authors the bar and self-declares again, and is falsified again |

**On recurrent bar-falsification, invert verdict authority:** after a second operator falsification, the loop stops declaring success at all — completion becomes an evidence pack (one exhibit per bar clause, pre-named inventories closed by name) presented to the operator, who alone rules. If the falsifications were coverage-shaped, pair with a coverage-first standing order: optimizing inside current coverage while named gaps stand is itself a collapse (basining).

**The brief must refuse to encode any of these as a success condition.** They are diagnostics for a running loop, and a refuse-list for the author.
